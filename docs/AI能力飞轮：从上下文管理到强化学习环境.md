# AI 能力飞轮：从上下文管理到强化学习环境

这篇记录一条关于 AI 能力提升的思考链：

```text
模型发布
→ 暴露能力缺陷
→ 工具和 Agent 框架补缺陷
→ 产生真实任务轨迹
→ 轨迹用于训练模型
→ 缺陷被内化
→ 暴露新缺陷
→ 继续循环
```

工具和 Agent 框架短期补能力，长期产出训练数据。

## 1. 语言空间的多尺度结构

语言和世界知识具有多尺度、层级、长尾、幂律、局部低维等特征。

```text
语言是非均匀空间
→ 存在多尺度可压缩结构
→ 神经网络通过拟合学习这些结构
→ 模型规模扩大后覆盖更深层、更稀疏的规律
→ 部分能力在评测上表现为涌现
```

“分形”可以作为直觉。核心概念是多尺度可压缩结构。

## 2. 神经网络的核心能力

神经网络通过拟合学习可泛化表征。

关键能力：

- 学习输入到输出的映射；
- 在中间层形成特征空间；
- 把数据规律压缩进参数；
- 把已学模式组合到新场景；
- 在巨大参数空间中找到可泛化的低损失区域。

GPT 是序列建模特化的神经网络。它强化了语言、代码、对话、上下文学习和工具调用。

对应弱点：精确感知、连续控制、严格计算、持久在线学习、现实因果校验。

## 3. 权重是慢变量，KV cache 是快变量

有用的类比：

```text
权重 ≈ 长期记忆 / 慢变量
KV cache / context ≈ 短期状态 / 快变量
```

权重包含记忆、算法结构、先验能力和世界统计。KV cache 是当前上下文产生的运行态。

自回归 Transformer 的运行可以写成：

```text
固定参数: theta
当前状态: KV_cache_t
输出: p(next_token | theta, KV_cache_t)
状态更新: KV_cache_{t+1} = KV_cache_t + new_KV
```

LLM 的状态分布在权重、上下文、KV cache、外部记忆和工具环境里。关键问题是快变量和慢变量之间的交换、压缩、巩固、重写。

## 4. 上下文管理正在被模型内化

KV cache 原本主要是工程缓存，用来避免重复计算历史 token。现在它开始成为模型工作记忆接口。

模型可以被训练去决定：

- 保留什么；
- 丢弃什么；
- 压缩什么；
- 何时检索；
- 何时写入长期记忆；
- 是否用隐向量替代显式文本；
- 如何重构当前任务状态。

这会吸收很多 Agent 框架的认知逻辑：上下文管理、任务拆解、反思、重试、状态维护。

## 5. Agent 框架的角色变化

主动上下文管理做成后，Agent 框架的重要性会变形。

旧 Agent 框架偏外部认知架构：

```text
规划
反思
重试
工具路由
上下文压缩
长期记忆
任务状态管理
```

新 Agent 框架偏运行时和操作系统：

```text
权限
沙箱
工具协议
状态持久化
日志
审计
部署
企业系统集成
```

框架从“帮模型思考”转向“让模型安全、稳定、可控地行动”。

## 6. 工具轨迹反过来训练模型

代码 Agent 的轨迹：

```text
生成 patch
→ 运行测试
→ 失败
→ 读错误
→ 修改
→ 再测试
→ 成功
→ 保存轨迹
→ 训练模型
```

这类轨迹包含目标、行动、环境反馈、失败恢复、工具调用、最终验证。

互联网文本记录“人类说过什么”。Agent 轨迹记录“模型如何把事情做成”。

工具把模型带进真实任务环境，并把补缺陷的过程变成训练材料。

## 7. 可验证任务可以构造成强化学习环境

存在可验证成功轨迹分布的能力，有机会被训练出来。

条件：

```text
可判定成功
+ 可探索动作空间
+ 足够多成功/失败轨迹
+ 反馈能支持信用分配
= 可以通过 RL、搜索、蒸馏、行为克隆内化能力
```

常见组合：

- 行为克隆成功轨迹；
- SFT 高质量过程；
- 搜索后蒸馏；
- 用可验证奖励做 RLVR；
- 用过程奖励模型评价中间步骤；
- 用失败轨迹做偏好或反例训练。

能力可以定义成：

```text
在某个环境和反馈函数下，
稳定产生成功轨迹的策略分布。
```

训练目标：把“偶尔跑通”变成“高概率、低成本、可泛化地跑通”。

## 8. 强反馈领域先形成飞轮

强反馈领域的特点：成功能判定，失败有信息，动作可以反复尝试。

典型领域：

- 代码：测试、编译、lint、benchmark；
- 数学：答案或证明可验证；
- 游戏：胜负和分数；
- 定理证明：proof checker；
- 数据库：查询结果是否匹配；
- 网页任务：目标状态是否达成；
- 工具调用：API 返回是否满足约束。

闭环：

```text
尝试
→ 反馈
→ 修正
→ 成功
→ 轨迹沉淀
→ 训练
```

代码、数学、工具调用、数据分析、浏览器操作会先受益。

## 9. 弱反馈领域的边界

弱反馈领域：

- 战略和管理；
- 产品设计；
- 教育；
- 慢病和心理健康；
- 法律与政策；
- 人际关系和谈判；
- 创意与审美；
- 科研选题；
- 金融投资；
- 伦理判断。

这些任务的反馈弱、慢、主观、难归因。

例子：

- 战略选择几年后才知道结果，外部环境强干扰；
- 产品改版可能提高点击率，同时损害长期信任；
- 教育方案可能提高分数，同时损害理解和兴趣。

弱反馈领域适合让 AI 做：

- 暴露假设；
- 列出风险；
- 提供备选方案；
- 拆解可验证子问题；
- 对比案例；
- 辅助专家判断；
- 记录长期反馈。

强反馈领域训练模型“做对”。弱反馈领域训练模型“想得完整，问得准确，把风险摊开”。

## 10. 当前判断

压缩后的思考链：

```text
语言和世界有多尺度结构
→ 神经网络通过拟合学习可泛化表征
→ GPT 把能力特化到序列和上下文
→ 权重是慢变量，KV cache/context 是快变量
→ 模型开始学习主动管理快变量
→ Agent 框架的认知功能被吸收
→ 工具和运行时产生真实任务轨迹
→ 轨迹用于 RL、搜索、蒸馏
→ 强反馈领域率先形成能力飞轮
→ 弱反馈领域仍依赖人类判断和长期评估
```

AI 能力提升会越来越像闭环系统：

```text
模型行动
环境反馈
工具扩大行动空间
轨迹沉淀为训练数据
训练更新模型
更强模型进入更复杂环境
```

核心变化：AI 从静态文本学习，扩展到从自己、工具、环境和人类共同产生的行动轨迹中学习。
