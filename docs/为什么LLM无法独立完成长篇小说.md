# 为什么当前 LLM 难以独立完成高一致性的长篇小说

在没有人工参与的情况下，只靠当前主流 LLM 稳定写出十万字以上、长期保持高一致性的小说，极其困难。

核心原因是当前主流自回归语言模型范式下的一类结构性限制。

长篇小说要求系统长期维护一个不断演化的世界状态。角色的性格弧线要在几十万字中保持连贯，情节的因果链条要前后自洽，第五章埋下的伏笔要在第三十章被自然地呼应。这些要求的本质，是对复杂叙事状态的长期精确维护。当前主流 LLM 缺乏一个可工程化、可持久维护的任务级世界状态机制。

## 当前的主流做法

由于 LLM 单次输出长度有限，长篇小说只能逐章节生成。每生成一个新章节，都需要向模型提供此前故事的背景信息作为上下文。围绕这个基本约束，目前主要有三种技术路线。

**路线一：逐层细化大纲。** 从一个高层故事构想出发，先生成粗纲，再拆分为细纲，最后逐章展开为正文。写手在展开某一章时，通常只能看到局部细纲，缺少故事全貌。大纲提供结构骨架，角色的微妙变化、场景之间的情感承接、前文细节在后文中的回响，很多都存在于骨架之外。细纲里写着"李明在本章与张华产生矛盾"，矛盾的质感、分寸、以及它和前文十几个场景之间的情感关联，很难被一行大纲承载。

**路线二：多 Agent 角色分工。** 设立多个专门化的 Agent——世界观构建者、导演、写手、审查员——各司其职，协同完成创作。核心瓶颈在审查员：它需要维持一个精确的故事事实状态，检查新内容是否与已有设定矛盾。审查员本身也是 LLM，同样难以可靠追踪一个在十万字中不断变化的世界。让一个有同样缺陷的模型去审查另一个模型，只会推迟问题。

**路线三：模拟世界。** 构建一个由多个 Agent 扮演角色的虚拟小镇，让角色在模拟环境中自主行动、对话、产生事件，然后将发生的事件转写为小说文本。每个 Agent 的记忆和决策都由 LLM 驱动，长期运行后，角色行为容易偏离初始设定。那个"安静内向的图书管理员"可能在第 30 天突然变成了社交达人，系统缺少稳定察觉这种漂移的机制。

三条路线都卡在同一个地方：**无法构建一个可靠的、可供 LLM 稳定使用的任务级叙事状态。**

## 实践中的真实困境

下面这些问题来自实践层面的稳定观察。

**显式提示的锚定效应。** 当上下文提供详细背景信息——角色关系、未解决的冲突、需要推进的情节线——模型容易进入"完成任务"模式。你写明"本章需要体现李明对张华的不信任"，模型就会在每一段对话、每一个动作里塞进不信任的信号，像一个过于用力的演员。背景越具体，锚定越严重；模型的注意力被显式信息吸走，自然展开、制造意外、留白和暗示的能力下降。

**角色塑造的两难困境。** 在提示词中显式描述角色性格（"王芳：性格倔强，重情义，说话直接"），模型会机械执行这些标签。每句对话都在喊"我很倔强"，角色变成属性标签的拟人化。改用隐式方式，通过前文的行为和对话让模型"理解"角色，跨几个章节后又容易漂移。上一章还沉默寡言的人，这一章突然变得话多起来，因为模型已经"忘了"这个角色是谁。显式描述导致死板，隐式暗示导致漂移。

**章节的模板化。** 还有一个更隐蔽的问题：由于模型的单次输出长度有限，每个章节的生成是一次独立的请求。而 LLM 有一个根深蒂固的倾向——把每次输出写成一个结构完整的小故事，有起承转合，有情感闭环。十个章节读下来，每一章都像一篇独立的短篇小说，有着几乎相同的叙事节奏。长篇小说应有的那种跨章节的张力蓄积、节奏变化、留白与爆发的交替，在这种逐章生成的模式下几乎不可能实现。

这些问题可以通过更精巧的提示工程局部缓解。困难在于同时解决：削弱角色脸谱化，容易加剧性格漂移；试图打破章节模板化，又会引入新的信息锚定问题。这些症状指向同一个深层原因。

## 根本原因

上述这些现象——范式层面的失败和实践中的具体困境——可以归结为三个相互关联、层层递进的技术原因。

### 上下文腐烂

LLM 处理信息的唯一通道是文本上下文。无论是直接拼接前文、还是通过 RAG 检索相关片段、还是用摘要压缩历史内容，最终都要以文本的形式进入模型的注意力窗口。

上下文是有损的信息通道。当上下文变长，模型对信息的提取能力呈现出一种被称为"U 型崩溃"的模式：对开头和结尾的信息保持较好的注意力，对中间大量内容的处理能力急剧下降。Liu et al. (2023) 的论文 *Lost in the Middle: How Language Models Use Long Contexts* 系统地验证了这一点。

把一部小说的前文信息塞入上下文——角色关系的变化、早期埋下的伏笔、次要情节线的进展——其中大量内容实质上处于模型的"盲区"。许多信息在注意力机制层面没有被有效处理。

压缩会把问题换成另一种形式。为了应对上下文长度限制，常见做法是将前文压缩为摘要。摘要本身是有损压缩，必然丢失细节。哪些细节被保留、哪些被丢弃，取决于执行摘要的 LLM 在那个时刻的"判断"。一个在第三章显得无关紧要的细节，可能在第二十章成为关键情节转折的基础；它在早期摘要中被丢弃后，就从系统的"记忆"中消失。

上下文越长，信息通道越不可靠。你往里塞的信息越多，能被准确提取的比例越低。

### 误差的滚雪球

误差累积是信息损耗的动态后果。

自回归模型逐 Token 预测，每一个新 Token 都基于前面所有 Token 生成，每一步都有微小的错误概率。Yann LeCun 近年反复用一个简单估算说明长程自回归生成的脆弱性：如果每个 Token 的错误概率为 `e`，生成 `N` 个 Token 后整体正确的概率为 `(1-e)^N`。对于十万字级别的文本——大约 15 到 20 万 Token——即使 `e` 极小，这个值也会快速下降。这个估算可以说明长链条中微小偏差的累积问题。

错误还会自我强化。Zhang et al. (2023) 在 *How Language Model Hallucinations Can Snowball* 中揭示了一个关键机制：当 LLM 在前文犯下一个微小的事实错误后，它往往不会在后续生成中自动修正。为了维持上下文的表面连贯性，模型会倾向于与前文错误保持一致，实质上是在"圆谎"。一个小错误引出配套的新细节，新细节又引出更多偏离事实的内容。这篇工作主要讨论幻觉，但为长篇叙事中的误差级联提供了旁证。

AI 生成的长篇小说常出现特征性的退化曲线：前几章质量尚可，中段开始出现轻微不一致，后期逐步失控。前半段累积的微小偏差，经过十几万 Token 的传递和放大，会严重到无法忽视。

### 缺乏外部纠错机制

第三个原因是缺乏及时纠错机制。

在代码生成任务中，LLM 有一个巨大的结构性优势：编译器和测试用例提供了硬性的外部反馈。代码要么能跑，要么不能跑；测试要么通过，要么不通过。这种二值化的反馈信号与模型本身完全独立，能够有效地截断误差的累积链条。即使模型犯了错，编译失败或测试不通过的信号会迫使系统修正，阻止错误向下游蔓延。

小说没有编译器。

多 Agent 方案通常会让另一个 LLM 充当"审查员"，检查生成内容与已有设定之间的一致性。Kambhampati (2024) 在 ICML 论文 *LLMs Can't Plan, But Can Help Planning in LLM-Modulo Frameworks* 中强调：系统缺乏独立于 LLM 的外部验证器时，单靠 LLM 之间互相审查，很难形成稳定的负反馈闭环。

用 LLM 审查 LLM 的生成内容，相当于用一个有同样认知缺陷的系统去校验另一个系统。审查 LLM 也会受到上下文腐烂的影响，也会在长文本中丢失关键信息，也会被表面的连贯性欺骗。它可以发现一些明显矛盾（角色在同一章里名字前后不一致），对需要跨越几十章、追踪多条情节线才能发现的深层不一致——比如角色动机的悄然转变、因果链条的断裂——检测能力有限。

代码世界有编译器作为独立于模型的客观裁判。语义世界里没有这样的裁判。没有一个外部系统能自动判定"这个角色在第 37 章的行为是否与第 5 章的性格设定一致"。在当前的技术条件下，能较可靠地做出这种判断的，仍然主要是人类读者。

三层原因叠加成闭合困局：上下文腐烂导致信息传递有损，有损的信息引发误差累积，累积的误差缺少外部机制纠正。这是当前范式下的结构性限制。

## 缺失的那块拼图

综合以上分析，可以得出一个更精确的判断：

在当前主流 LLM 范式下，长篇小说生成系统如果主要依赖文本上下文来读取、压缩、更新和执行叙事状态，长期一致性控制能力会随着文本长度增长而系统性退化。长上下文窗口、外部记忆检索、LLM 自我修订，可以延缓退化速度，很难从根本上阻止退化。

问题的根源是：这些系统缺少一个**可工程化的、可更新的任务级世界状态机制**，也就是人类作家在写作时脑海中自然维持的东西。

一个小说家在创作长篇时，脑子里始终运转着一个关于故事世界的内部模型。它包含角色之间的张力、情节的走向、尚未兑现的线索、当前的情感基调，并随着每一章的写作自然更新，是叙事决策的隐性基础。

谢赛宁在描述世界模型时给出过一个精确的定义：给定一个系统的当前状态 `s_t` 和一个动作 `a_t`，世界模型是一个能预测下一个状态 `s_{t+1}` 的转移函数。这个状态是系统的充分统计量——它保留对未来预测和决策所需的关键信息，同时忽略无关细节。而构建这样的状态，本质上是一个表征学习问题。

这个状态可以是显式结构，也可以是特征空间中的隐藏表示（latent representation），可以是层次化的、抽象的。就像我们不需要模拟每一个空气分子的碰撞来理解天气，故事的世界状态也不需要把每一个细节都用自然语言写出来。它需要在某个表征空间中编码故事世界的关键信息——角色关系、情节因果、未解决的冲突、情感张力——并随着故事发展持续更新。

LLM 已经学到了大量关于现实与叙事的统计规律。当前主流 LLM 缺乏一个可持久、可读写、可验证、适合长程任务控制的世界状态机制。它对故事世界的理解散落在文本上下文和隐式表征中，每次生成都要从文本片段中临时重建局部状态。这个过程一次性、有损、容易走样：上下文窗口里的信息可能被忽视，检索回来的段落可能不完整，压缩摘要必然丢失细节，LLM 的自我修订可能引入新的错误。缺失的是一个能被稳定维护和持续更新的任务级世界状态接口。

## 结语

笔者在过去相当长的时间里反复尝试过各种流程编排——调整大纲粒度、改进摘要策略、增加审查环节、精细控制提示词。上面提到的问题都真实地、反复地出现过。很长一段时间里，我把核心问题理解成记忆的压缩、保存和检索：让模型更准确地"记住"前文。

后来逐渐意识到，记忆压缩的失真、检索的遗漏、角色的漂移、章节的模板化，都指向同一个缺失。谢赛宁对世界模型的描述让我把这个想法想清楚：一个模型应该拥有一个可更新的世界状态，这个状态是系统的充分统计量，包含对未来预测和决策所需的关键信息，并且可以在特征空间中以隐藏表征的形式存在。这和我在小说生成中反复碰壁后得出的直觉吻合：小说生成系统需要一个能随故事发展持续更新的世界状态机制。当前 LLM 拥有隐式的世界知识，但缺少一个可工程化的长期叙事状态层。

增大上下文窗口或设计更巧妙的 Agent 流程只能缓解问题。当前的 LLM 把故事的"状态"主要寄托在文本上下文上，相当于一个作家每写一章都要靠前文片段、摘要和提示词来重新回忆故事世界。这种方式可以工作一段时间，随着篇幅增长，关键而低频的叙事状态越来越容易丢失、走样、漂移。

解决方向可能是让模型拥有一个可更新的内部世界状态。这个状态可以是显式数据库、知识图谱，也可以是潜在空间中的某种表征；它需要随着叙事推进持续演化，编码故事世界中真正重要的信息，并可靠指导后续生成。
